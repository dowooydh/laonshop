# LAON SHOP QA 운영 구조

이 폴더는 Codex QA/테스트 담당 세션의 작업 공간이다. 제품 코드는 사용자가 명시적으로 허락하기 전까지 수정하지 않고, 테스트 시나리오, 실행 기록, 리뷰 결과, 재현 문서만 이곳에 남긴다.

## 현재 구성

- `inventory/`: 프로젝트 구조, 기능 목록, 테스트 대상 분석
- `checklists/`: 수동/E2E/회귀 테스트 체크리스트
- `data/`: 테스트 데이터 정책, 정리 기준, 금지 작업
- `handoff/`: 최신 QA 결과와 개발 세션 전달 문서
- `e2e/`: 향후 Playwright 등 E2E 테스트 코드 위치
- `api/`: 향후 API/서버 액션/PG 콜백 테스트 코드 위치
- `reports/`: 실행별 원본 로그, 스크린샷, 증적 위치

## QA 원칙

1. 실제 사용자 관점으로 흐름을 검증한다.
2. 카드사 심사 필수 요건을 별도 축으로 관리한다.
3. 결제 금액은 DB/서버 기준으로 검증하고, 클라이언트 장바구니 금액은 신뢰하지 않는다.
4. 카드정보, PG 키, 세션 시크릿, 운영 마스터 데이터는 테스트 산출물에 남기지 않는다.
5. 결함을 발견하면 제품 코드 수정 대신 재현 경로, 원인 추정, 관련 파일, 우선순위를 `handoff/latest.md`에 정리한다.

## 테스트 요청 시 기본 절차

1. 최근 변경사항 확인: `git status`, `git diff`, 필요 시 최근 커밋 비교
2. 영향 범위 분석: 라우트, 서버 액션, DB 모델, PG 경계, UI 흐름
3. 시나리오 선정: happy path, 취소/뒤로가기, 중복 요청, 권한, 빈 상태, 에러 상태, 모바일
4. 실행 및 증적 수집: 브라우저, API, DB 확인, 콘솔/서버 로그 확인
5. 정리: 테스트 데이터 청소 원칙 준수, `handoff/latest.md` 갱신
