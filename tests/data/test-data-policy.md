# 테스트 데이터 정책

## 생성 가능

- `qa+YYYYMMDD-HHMM@example.test` 형식의 테스트 회원
- 테스트 회원이 직접 생성한 주문
- 테스트 회원 브라우저의 localStorage 장바구니
- 로컬/테스트 DB의 임시 상품. 단, 사전 합의된 seed 또는 QA 전용 prefix를 사용한다.

## 삭제 가능

- QA 세션이 직접 만든 테스트 회원과 그 주문
- QA 세션이 직접 만든 QA 전용 상품
- 브라우저 localStorage, session cookie
- 테스트 실행 산출물(`tests/reports/` 내부)

## 삭제 금지

- 운영 회원
- 운영 주문
- 운영 상품 마스터
- PG/Vercel/DB 환경설정
- `.env`, `.env.local`, PG 상점키, 세션 시크릿
- 카드번호, 유효기간, 생년월일, 비밀번호 앞자리 등 카드정보

## 데이터 정리 원칙

1. 정리 전 대상이 QA 생성 데이터인지 식별한다.
2. 주문/주문상품 삭제는 테스트 사용자 범위로 제한한다.
3. 운영 DB인지 불명확하면 삭제하지 않는다.
4. 테스트 증적에는 민감정보를 마스킹한다.
5. KSPAY 테스트 거래는 PG 정책에 따라 자동취소 여부와 시간을 기록한다.

## 권장 테스트 계정

- 이메일: `qa+<date>-<case>@example.test`
- 이름: `QA테스트`
- 전화번호: `010-0000-0000`
- 배송지: `서울특별시 테스트구 테스트로 1`
