# 수동 E2E 체크리스트

## 0. 사전 확인

- [ ] 현재 브랜치와 최근 변경사항 확인
- [ ] Node 22.x, pnpm 11.5.3 확인
- [ ] `.env`에 `DATABASE_URL`, `DIRECT_URL`, `SESSION_SECRET`, `PG_MODE=kspay`, `KSPAY_STORE_ID`, `SHOP_APP_URL` 존재 확인(값은 출력하지 않음)
- [ ] 테스트 DB/로컬 DB인지 확인
- [ ] `pnpm prisma db push`, `pnpm db:seed` 완료 여부 확인
- [ ] `pnpm build` 또는 요청 범위에 맞는 검증 명령 통과 여부 확인
- [ ] 테스트 데이터 정리 범위 확인
- [ ] Vercel Preview가 아닌 로컬 `http://localhost:3003`에서 결제 테스트하는지 확인

## 1. 홈/상품 탐색

- [ ] `/` 홈 로드
- [ ] 헤더의 남성의류/여성의류 링크 동작
- [ ] `/shop/men`, `/shop/women` 상품 카드 노출
- [ ] 상품 이미지/상품명/가격 표시
- [ ] 카테고리 탭 동작
- [ ] 정렬 동작
- [ ] `/search` 검색 결과
- [ ] 검색 빈 결과 상태
- [ ] 잘못된 gender URL은 404
- [ ] 상품 상세 진입
- [ ] 상세에서 뒤로가기/홈 이동 가능
- [ ] 최근 본 상품이 상품 상세 방문 후 노출
- [ ] 품절 상품은 구매 불가 UI 표시

## 2. 장바구니

- [ ] 사이즈 기본 선택값 확인
- [ ] 수량 감소가 1 미만으로 내려가지 않음
- [ ] 장바구니 담기 후 localStorage 저장
- [ ] 같은 상품/같은 사이즈 추가 시 수량 합산
- [ ] 같은 상품/다른 사이즈 추가 시 별도 라인
- [ ] 장바구니 수량 증감 반영
- [ ] 장바구니 삭제 반영
- [ ] 전체 삭제 후 빈 상태 표시
- [ ] 새로고침 후 장바구니 유지
- [ ] 시드 재실행/죽은 상품 ID 케이스에서 판매 종료 오류 표시

## 3. 인증

- [ ] 비로그인 상태에서 `/checkout` 접근 시 로그인으로 이동
- [ ] 비로그인 상태에서 `/mypage` 접근 시 로그인으로 이동
- [ ] 약관 미동의 회원가입 오류
- [ ] 개인정보 수집·이용 미동의 회원가입 오류
- [ ] 회원가입 필수값 검증
- [ ] 비밀번호 8자 미만 오류
- [ ] 중복 이메일 오류
- [ ] 같은 이메일 동시 가입 2건 중 하나는 정상 중복 안내로 종료되고 500이 발생하지 않음
- [ ] 이메일 대소문자 변형이 동일 계정으로 취급되고 중복 가입되지 않음
- [ ] 회원가입 성공 후 로그인 상태 반영
- [ ] 로그아웃 후 보호 페이지 접근 차단
- [ ] 잘못된 비밀번호 로그인 오류
- [ ] 같은 이메일 5회 실패 시 10분 잠금
- [ ] 정상 로그인 후 헤더 사용자명 표시

## 4. 찜/최근 본 상품

- [ ] 로그인 사용자가 상품 상세에서 찜 추가
- [ ] 찜 토글 재클릭 시 제거
- [ ] 비로그인 찜 클릭 시 로그인 유도
- [ ] 마이페이지 찜 목록 표시
- [ ] 최근 본 상품 localStorage 최대 보관/노출 동작

## 5. 주문/결제

- [ ] 빈 장바구니 checkout 상태
- [ ] 마이페이지 기본 배송지 또는 최근 주문 배송지가 자동입력
- [ ] 배송정보 누락 시 오류
- [ ] 구매조건 동의 미체크 시 오류
- [ ] 카드/카카오/네이버/실시간계좌이체 KSPAY 수단 선택
- [ ] 원클릭 카드 등록 UI가 없고, stale oneclick 요청도 주문 생성 전 사용자 친화적 오류로 차단
- [ ] 수기결제는 API key + live switch 이중 가드 없이는 UI·서버 모두 비활성이고 mock PAID를 만들지 않음
- [ ] `reCommConId`/`reHash` 주문 결박 규격 전에는 테스트 MID `2999199999`만 서버승인하고 실 MID는 외부 승인 전에 차단
- [ ] 결제 버튼 중복 클릭 시 중복 제출 방지 확인
- [ ] 동일 checkout을 두 탭에서 동시에 제출해도 주문과 PG 승인이 한 번만 생성
- [ ] 동일 상품을 다른 사이즈 라인으로 나눠도 상품별 총수량으로 재고 검증
- [ ] 두 사용자가 마지막 재고를 동시에 주문해도 하나만 예약/승인
- [ ] 상품에 없는 사이즈를 서버 요청에 넣으면 거부
- [ ] 주문 생성 후 KSPAY 결제창 로딩 화면 표시
- [ ] jQuery가 KSPAY 스크립트보다 먼저 로드되는지 확인
- [ ] KSPAY callback 정상 케이스에서 result 제출 흐름 확인
- [ ] KSPAY 취소 케이스에서 현재 화면 복귀/상태 확인
- [ ] 결제 실패 시 주문 결과 FAILED 표시
- [ ] 결제 완료 시 주문 결과 PAID 표시 및 장바구니 비움
- [ ] 주문 완료 화면 영수증 링크 표시
- [ ] result 재호출 시 PENDING이 아닐 경우 중복 승인 처리 없음
- [ ] result 병렬 2회 호출 시 외부 승인 1회, 성공 상태가 실패 응답에 의해 소실되지 않음
- [ ] PG 500/503/timeout에서 주문 상태·재시도·운영 보정 경로 확인
- [ ] jQuery/KSPAY script 차단 시 오류 안내, 재시도, 주문 복귀 동작
