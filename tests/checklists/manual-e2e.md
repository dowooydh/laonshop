# 수동 E2E 체크리스트

## 0. 사전 확인

- [ ] 현재 브랜치와 최근 변경사항 확인
- [ ] `.env.local`에 `DATABASE_URL`, `DIRECT_URL`, `SESSION_SECRET`, `PG_MODE=kspay`, `KSPAY_STORE_ID` 존재 확인
- [ ] 테스트 DB/로컬 DB인지 확인
- [ ] `pnpm build` 또는 요청 범위에 맞는 검증 명령 통과 여부 확인
- [ ] 테스트 데이터 정리 범위 확인

## 1. 홈/상품 탐색

- [ ] `/` 홈 로드
- [ ] 헤더의 남성의류/여성의류 링크 동작
- [ ] `/shop/men`, `/shop/women` 상품 카드 노출
- [ ] 상품 이미지/상품명/가격 표시
- [ ] 잘못된 gender URL은 404
- [ ] 상품 상세 진입
- [ ] 상세에서 뒤로가기/홈 이동 가능

## 2. 장바구니

- [ ] 사이즈 기본 선택값 확인
- [ ] 수량 감소가 1 미만으로 내려가지 않음
- [ ] 장바구니 담기 후 localStorage 저장
- [ ] 같은 상품/같은 사이즈 추가 시 수량 합산
- [ ] 같은 상품/다른 사이즈 추가 시 별도 라인
- [ ] 장바구니 수량 증감 반영
- [ ] 장바구니 삭제 반영
- [ ] 전체 삭제 후 빈 상태 표시
- [ ] 새로고침 후 장바구니 유지

## 3. 인증

- [ ] 비로그인 상태에서 `/checkout` 접근 시 로그인으로 이동
- [ ] 비로그인 상태에서 `/mypage` 접근 시 로그인으로 이동
- [ ] 회원가입 필수값 검증
- [ ] 비밀번호 6자 미만 오류
- [ ] 중복 이메일 오류
- [ ] 회원가입 성공 후 로그인 상태 반영
- [ ] 로그아웃 후 보호 페이지 접근 차단
- [ ] 잘못된 비밀번호 로그인 오류
- [ ] 정상 로그인 후 헤더 사용자명 표시

## 4. 주문/결제

- [ ] 빈 장바구니 checkout 상태
- [ ] 배송정보 누락 시 오류
- [ ] 계좌이체/정기결제/수기결제 버튼 비활성
- [ ] 결제 버튼 중복 클릭 시 중복 제출 방지 확인
- [ ] 주문 생성 후 KSPAY 결제창 로딩 화면 표시
- [ ] jQuery가 KSPAY 스크립트보다 먼저 로드되는지 확인
- [ ] KSPAY callback 정상 케이스에서 result 제출 흐름 확인
- [ ] KSPAY 취소 케이스에서 현재 화면 복귀/상태 확인
- [ ] 결제 실패 시 주문 결과 FAILED 표시
- [ ] 결제 완료 시 주문 결과 PAID 표시 및 장바구니 비움
- [ ] result 재호출 시 PENDING이 아닐 경우 중복 승인 처리 없음

## 5. 주문/마이페이지

- [ ] 본인 주문 상세 접근 가능
- [ ] 타인 주문 상세 접근 불가
- [ ] 없는 주문 ID는 404
- [ ] 마이페이지 주문 목록 최신순 표시
- [ ] 주문 상태 배지 표시
- [ ] 주문 내역 빈 상태 표시

## 6. 카드사 심사

- [ ] 푸터에 상호, 사업자등록번호, 통신판매업신고번호 문구, 대표/주소/연락처 요구사항 확인
- [ ] 이용약관 접근
- [ ] 개인정보처리방침 접근
- [ ] 청약철회/교환/환불 안내 접근
- [ ] 결제경로 캡처: 상품 상세 → 장바구니 → 주문/결제 → KSPAY 결제창

## 7. 반응형/에러

- [ ] 모바일 390px 주요 화면 레이아웃
- [ ] 데스크톱 1440px 주요 화면 레이아웃
- [ ] 텍스트 겹침/잘림 없음
- [ ] 키보드 포커스 기본 사용 가능
- [ ] 브라우저 콘솔 오류 확인
- [ ] 서버 로그 오류 확인
